## Proof Logic

The claim is disproved by consensus: three independent, peer-reviewed studies each find that AI-generated or AI-assisted code is **not** more secure than human-written code.

**B1 — Pearce et al. 2022 (IEEE S&P):** The first landmark study systematically tested GitHub Copilot across 89 scenarios targeting the MITRE CWE Top-25 most dangerous software weaknesses. Generating 1,689 programs, the researchers found approximately 40% to be vulnerable (B1). While these prompts were security-sensitive by design, they cover real-world coding patterns that developers actually encounter. This study established that AI code generation tools can introduce vulnerabilities at high rates in security-relevant contexts.

**B2 — Perry et al. 2023 (ACM CCS):** A controlled user study directly compared programmers with and without access to an AI coding assistant. Participants with AI access "wrote significantly less secure code than those without access to an assistant" (B2). This is important because it tests the real-world usage pattern — developers using AI to help write code — and still finds degraded security outcomes. The study also found a "false sense of security": AI-assisted participants were more likely to believe their code was secure even when it was not.

**B3 — Cotroneo et al. 2025 (IEEE ISSRE):** The largest-scale study to date, comparing over 500,000 Python and Java code samples from human developers against code generated by three modern LLMs (ChatGPT, DeepSeek-Coder, Qwen-Coder). Using Common Weakness Enumeration (CWE) classification, the study found that "AI-generated code also contains more high-risk security vulnerabilities" (B3). Earlier reporting on this study identified AI code as 1.88x more likely to introduce improper password handling, 1.91x more likely to make insecure object references, and 2.74x more likely to add cross-site scripting (XSS) vulnerabilities.
