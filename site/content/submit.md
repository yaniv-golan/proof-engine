# Submit a Proof

Share your verified proofs with the community. Submissions are reviewed and published automatically once merged.

## Before You Submit

- Your proof must include all four artifacts: `proof.py`, `proof.md`, `proof_audit.md`, `proof.json`
- `proof.py` must produce a valid JSON summary block when executed
- `validate_proof.py` must return PASS (this is a heuristic structural check, not an exhaustive rule-compliance certificate — but it catches common issues)
- Generated by Proof Engine v0.9.0 or later (must include `generator` block)

## Steps

1. **Fork & clone the repo**
   ```
   gh repo fork yaniv-golan/proof-engine --clone
   ```

2. **Run your proof and save the JSON summary**
   ```
   python proof.py > output.txt
   # Extract the JSON block after "=== PROOF SUMMARY (JSON) ===" and save as proof.json
   ```

3. **Add your proof**
   Copy all four files to `site/proofs/your-claim-slug/`

4. **Optionally add meta.yaml**
   ```yaml
   tags: [economics, inflation]
   featured: false
   ```
   If omitted, tags are auto-assigned by keyword matching.

5. **Validate locally**
   ```
   python proof-engine/skills/proof-engine/scripts/validate_proof.py site/proofs/your-claim-slug/proof.py
   ```

6. **Open a PR**
   CI runs validation automatically. A maintainer reviews and merges. Your proof goes live on the next deploy.
